# Vionix 文档中心

本文档中心按“产品需求、设计方案、开发基线、审查记录”组织，避免设计资料、开发说明和部署资产混放。

## 文档树

```text
docs/
├── README.md
├── design/          # 架构与专题设计方案
├── development/     # 研发、测试、部署、验收基线
├── prd/             # 产品需求与版本需求预留
└── review/          # 文档审查和设计审查记录
```

## 文档列表

| 目录 | 说明 | 入口 |
|------|------|------|
| `docs/design/` | 项目总览、系统架构、基础设施、RBAC、规则引擎、前端、低代码仪表盘等设计方案 | [设计文档索引](design/README.md) |
| `docs/development/` | 需求规格、总体设计、详细设计、接口、数据、前端、部署、测试、计划、协作规范 | [开发文档总览](development/README.md) |
| `docs/prd/` | 后续产品需求、版本规划、需求变更记录 | [PRD 目录](prd/README.md) |
| `docs/review/` | 设计与开发文档审查报告 | [项目设计方案审查报告](review/项目设计方案审查报告.md) |

## 维护规则

1. 产品目标变化先更新 `docs/prd/`，再同步设计和开发文档。
2. 架构、数据、接口或部署变化先更新 `docs/design/` 和 `docs/development/` 对应文档。
3. 可运行部署资产放在 `deploy/`，文档中只保留说明和链接。
4. 每次实现交付前检查 README 中的目录树是否仍与仓库一致。
