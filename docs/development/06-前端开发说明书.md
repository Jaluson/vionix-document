# Vionix 前端开发说明书

## 1. 前端目标

前端提供登录、实时监控、历史趋势、设备目录、设备分组、规则管理、告警中心、RBAC 管理和低代码仪表盘功能。安全边界由后端控制，前端负责交互、展示、订阅管理和用户体验。

## 2. 技术基线

| 项 | 技术 |
|----|------|
| 框架 | Vue 3 |
| 路由 | Vue Router |
| 状态 | Pinia |
| 图表 | ECharts |
| HTTP | Axios |
| 实时 | STOMP over WebSocket |
| 低代码布局 | vue-grid-layout 或等价方案 |

## 4. 路由设计

| 路由 | 页面 | 权限 |
|------|------|------|
| `/login` | 登录 | 匿名 |
| `/monitor` | 实时监控 | `menu:monitor` |
| `/devices` | 设备目录 | `menu:device` |
| `/alerts` | 告警中心 | `menu:alert` |
| `/rules` | 规则管理 | `menu:rule` |
| `/device-groups` | 设备分组 | `menu:device-group` |
| `/dashboards` | 仪表盘列表 | `menu:dashboard` |
| `/dashboards/:id` | 仪表盘查看 | 仪表盘查看权限 |
| `/dashboards/:id/edit` | 仪表盘编辑 | 创建者或租户管理员 |
| `/rbac/users` | 用户管理 | `menu:user` |
| `/rbac/roles` | 角色管理 | `menu:role` |
| `/tenants` | 租户管理 | 超级管理员 |

路由守卫根据登录状态和权限树控制访问。未授权路由返回 403 页面或重定向到可访问首页。

## 5. 认证状态

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| accessToken | 内存态或受控短期状态 | 减少 XSS 后持久泄露风险 |
| refreshToken | HttpOnly Cookie | 前端不可读 |
| userInfo | Pinia | 用户基础信息 |
| permissions | Pinia | 菜单、按钮、API 权限 |

Axios 拦截器职责：

1. 自动附加 `Authorization` header。
2. 处理 401，尝试调用 refresh。
3. refresh 失败后清理状态并跳转登录。
4. 不记录 token 到日志。

## 6. 实时监控页面

页面加载流程：

1. 通过 `/api/devices` 或 `/api/dashboard-vars/devices` 获取当前用户可见且启用设备列表。
2. 建立 WebSocket 连接并完成鉴权。
3. 按选中设备订阅实时指标 topic。
4. 并行请求历史数据填充分钟、小时、天级图表。
5. 请求 `/api/alerts/firing` 更新告警角标。

设备切换流程：

1. 取消旧设备 topic 订阅。
2. 清空实时曲线缓存。
3. 订阅新设备指标和告警 topic。
4. 重新请求历史趋势。

## 7. 设备目录页面

设备目录页面用于维护最小设备元数据和状态，不承载完整设备生命周期。

| 功能 | 要求 |
|------|------|
| 列表 | 支持关键字、状态、来源过滤，默认只展示当前租户内设备 |
| 新增/编辑 | 维护 `deviceId`、名称、来源、位置和 metadata |
| 状态 | 支持启用、禁用、归档；禁用后不能订阅实时数据或作为新规则目标 |
| 权限 | 设备管理按钮按权限展示，后端仍执行最终校验 |
| 变量复用 | 仪表盘和监控页设备下拉必须复用设备目录的可见设备语义 |

## 8. WebSocket 客户端

WebSocket store 需要维护：

| 状态 | 说明 |
|------|------|
| `connected` | 连接状态 |
| `sessionId` | 当前连接标识 |
| `subscriptions` | topic 到 subscription id 映射 |
| `retryCount` | 重连次数 |
| `lastError` | 最近错误 |

实现要求：

1. token 过期后先 refresh，再重连。
2. 订阅失败时展示明确错误。
3. 页面离开时取消页面级订阅。
4. 组件销毁时释放定时器和图表实例。

## 9. 图表和数据展示

| 场景 | 数据源 | 策略 |
|------|--------|------|
| 秒级实时曲线 | WebSocket | 追加点，默认保留最近 120 个点 |
| 分钟趋势 | `/api/metrics?level=min` | 定时刷新或手动刷新 |
| 小时趋势 | `/api/metrics?level=hour` | 按时间范围刷新 |
| 天趋势 | `/api/metrics?level=day` | 长周期统计 |
| 告警角标 | WebSocket + `/api/alerts/firing` | 首次查询，后续事件更新 |

图表组件必须处理空数据、加载中、错误、权限不足和断线重连状态。

## 10. 规则管理页面

规则编辑器要求：

1. 条件按 `groupIndex` 分组展示。
2. 组内条件为 AND，组间为 OR。
3. 不暴露 `logic_op` 字段。
4. `scope=DEVICE` 时选择设备；`scope=GROUP` 时选择设备分组。
5. 提交前校验指标、操作符、阈值、持续时间和动作配置。

## 11. 仪表盘编辑器

页面结构：

```
左侧组件库 -> 中央网格画布 -> 右侧配置面板
顶部变量栏 -> 控制 device、timeRange 等全局变量
```

组件配置示例：

```json
{
  "id": "w_001",
  "type": "line-chart",
  "title": "温度实时曲线",
  "grid": { "x": 0, "y": 0, "w": 8, "h": 3 },
  "datasource": {
    "metric": "temperature",
    "device": "$device",
    "level": "realtime",
    "agg": "mean"
  },
  "config": {
    "maxPoints": 120
  }
}
```

仪表盘组件只能调用已有监控接口和 WebSocket，不新增专用时序查询协议。

## 12. 前端安全要求

1. 不在 localStorage 长期保存 refreshToken。
2. 不把 token 输出到日志、错误弹窗或 URL。
3. 前端权限隐藏只用于体验，后端必须再次鉴权。
4. 渲染后端返回内容时避免直接 `v-html`。
5. WebSocket query token 只允许在 HTTPS/WSS 下作为备选方案。
6. 仪表盘 JSON 需要做 schema 校验，防止注入非法组件类型或危险配置。
