# Vionix 接口协议说明书

## 1. 通用约定

### 1.1 Base URL

```
/api
```

### 1.2 认证

HTTP 请求使用：

```
Authorization: Bearer {accessToken}
```

refreshToken 通过 HttpOnly Cookie 传递，不在 JavaScript 中读取。

### 1.3 统一响应

```json
{
  "code": "OK",
  "message": "success",
  "data": {},
  "traceId": "01HX..."
}
```

分页响应：

```json
{
  "code": "OK",
  "data": {
    "items": [],
    "pageNum": 1,
    "pageSize": 20,
    "total": 100
  }
}
```

错误响应：

```json
{
  "code": "FORBIDDEN",
  "message": "permission denied",
  "traceId": "01HX..."
}
```

常用错误码：

| code | HTTP | 场景 |
|------|------|------|
| `OK` | 200 | 成功 |
| `BAD_REQUEST` | 400 | 参数错误 |
| `UNAUTHORIZED` | 401 | 未认证或 token 失效 |
| `FORBIDDEN` | 403 | 无权限或数据范围不允许 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `CONFLICT` | 409 | 唯一约束或状态冲突 |
| `DEVICE_DISABLED` | 409 | 设备被禁用，不能订阅或作为新目标 |
| `TENANT_MISMATCH` | 403 | 请求目标租户与当前会话不一致 |
| `INTERNAL_ERROR` | 500 | 未预期服务端异常 |

### 1.4 租户约定

普通请求的 `tenant_id` 来自 JWT，不接受请求参数覆盖。超级管理员跨租户操作使用受控的 `X-Tenant-Id`，并记录审计日志。

### 1.5 HTTP 状态码

| 状态码 | 场景 |
|--------|------|
| `200` | 查询或更新成功 |
| `201` | 创建成功 |
| `400` | 参数格式、枚举、范围校验失败 |
| `401` | 未认证、token 缺失或过期 |
| `403` | 无权限或数据范围不允许 |
| `404` | 资源不存在或不应暴露存在性 |
| `409` | 唯一约束冲突或状态冲突 |
| `500` | 服务端异常 |

### 1.6 版本兼容

本期 API 以 `/api` 为基础路径。破坏兼容的接口变更必须通过发布说明标记；如果需要并行支持多个客户端版本，再引入 `/api/v2` 版本路径或请求 header 版本协商。

## 4. 设备目录接口

设备目录是监控、规则、告警、仪表盘变量和 WebSocket 订阅的数据权限来源。

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/devices` | 创建设备 |
| PUT | `/api/devices/{deviceId}` | 更新设备名称、来源、位置和 metadata |
| PUT | `/api/devices/{deviceId}/status` | 启用、禁用或归档设备 |
| GET | `/api/devices` | 当前用户可见设备列表 |
| GET | `/api/devices/{deviceId}` | 设备详情 |

创建请求：

```json
{
  "deviceId": "sensor-001",
  "name": "1号温度传感器",
  "source": "mqtt",
  "location": "1号车间",
  "metadata": {
    "model": "DHT22"
  }
}
```

列表查询参数：

| 参数 | 说明 |
|------|------|
| `keyword` | 按设备 ID 或名称模糊搜索 |
| `status` | `ENABLED|DISABLED|ARCHIVED` |
| `source` | `mqtt|modbus|http` |
| `pageNum` | 页码 |
| `pageSize` | 每页数量 |

权限约束：

1. 创建和更新设备需要设备管理权限。
2. 普通用户列表只返回数据范围内可见设备。
3. 禁用设备不能作为新规则目标、分组绑定或 WebSocket 订阅目标。
4. 已归档设备保留历史告警和时序数据可追溯性，但默认不进入变量选项。

## 5. 指标查询接口

### 5.1 历史指标

```
GET /api/metrics
```

参数：

| 参数 | 必填 | 说明 |
|------|------|------|
| `level` | 否 | `raw|min|hour|day`，不传可由后端按时间范围选择 |
| `measurement` | 否 | 默认 `device_metrics` |
| `fields` | 是 | 指标列表，如 `temperature,humidity` |
| `start` | 是 | 相对时间或 RFC3339 |
| `end` | 否 | 默认 `now()` |
| `agg` | 否 | `mean|sum|max|min|count`，默认 `mean` |
| `deviceId` | 否 | 设备 tag 过滤 |
| `source` | 否 | 数据来源 tag 过滤 |

响应：

```json
{
  "level": "min",
  "field": "temperature",
  "agg": "mean",
  "deviceId": "sensor-001",
  "data": [
    {"time": "2026-05-26T10:01:00Z", "value": 25.3}
  ]
}
```

服务端必须从 JWT 获取租户，并校验 `deviceId` 数据权限。

## 6. 规则和告警接口

### 6.1 规则管理

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/rules` | 创建规则，含 conditions 和 actions |
| PUT | `/api/rules/{id}` | 更新规则 |
| DELETE | `/api/rules/{id}` | 删除规则 |
| PUT | `/api/rules/{id}/toggle` | 启用或禁用规则 |
| GET | `/api/rules` | 分页查询规则 |
| GET | `/api/rules/{id}` | 规则详情 |

规则条件语义为组内 AND、组间 OR：

```json
{
  "name": "高温告警",
  "scope": "DEVICE",
  "targetId": "sensor-001",
  "severity": "CRITICAL",
  "conditions": [
    {
      "groupIndex": 0,
      "conditionType": "THRESHOLD",
      "metric": "temperature",
      "operator": ">",
      "threshold": 80,
      "durationSeconds": 300
    }
  ],
  "actions": [
    {
      "actionType": "HTTP",
      "config": {}
    }
  ]
}
```

### 6.2 告警

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/alerts` | 分页查询告警记录 |
| GET | `/api/alerts/firing` | 当前触发中告警 |
| GET | `/api/alerts/stats` | 告警统计 |

查询参数：`severity`、`status`、`deviceId`、`startTime`、`endTime`、`pageNum`、`pageSize`。

### 6.3 设备分组

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/device-groups` | 创建分组 |
| PUT | `/api/device-groups/{id}` | 更新分组 |
| DELETE | `/api/device-groups/{id}` | 删除分组 |
| GET | `/api/device-groups` | 分组列表 |
| POST | `/api/device-groups/{id}/devices` | 绑定设备 |
| DELETE | `/api/device-groups/{id}/devices` | 移除设备 |

绑定设备时必须校验设备属于当前租户且状态为 `ENABLED`。

## 8. WebSocket/STOMP 协议

连接：

```
CONNECT /ws
Authorization: Bearer {accessToken}
```

订阅：

| Topic | 消息 |
|-------|------|
| `/topic/tenant/{tenantId}/device/{deviceId}/metrics` | 单设备实时指标 |
| `/topic/tenant/{tenantId}/metrics` | 租户级实时指标 |
| `/topic/tenant/{tenantId}/alerts` | 租户级告警 |
| `/topic/tenant/{tenantId}/device/{deviceId}/alerts` | 单设备告警 |

实时指标消息：

```json
{
  "type": "METRIC_POINT",
  "tenantId": 1,
  "deviceId": "sensor-001",
  "source": "mqtt",
  "time": "2026-05-26T10:01:00Z",
  "metrics": {
    "temperature": 25.3,
    "humidity": 60.5
  }
}
```

告警消息：

```json
{
  "type": "ALERT_FIRING",
  "tenantId": 1,
  "data": {
    "alertId": 123,
    "ruleName": "高温告警",
    "severity": "CRITICAL",
    "deviceId": "sensor-001",
    "triggerValue": 92.5,
    "firedAt": "2026-05-26T10:01:00Z"
  }
}
```

## 9. MQTT 协议

生产 topic：

```
vionix/{tenant_id}/{device_id}/metrics
```

开发兼容 topic：

```
sensors/{device_id}
```

兼容 topic 只能用于单租户开发环境，生产必须能确定 `tenant_id`。

payload：

```json
{
  "timestamp": "2026-05-26T10:01:00Z",
  "source": "mqtt",
  "temperature": 25.3,
  "humidity": 60.5,
  "light": 320
}
```

数值字段进入降采样链路，非数值字段只用于状态或扩展。
