# Vionix 开发规范与协作流程

## 1. 仓库结构规范

目标完整结构：

```
vionix/
├── backend/
├── frontend/
├── mysql/
│   └── migrations/
├── influxdb/
├── mosquitto/
├── telegraf/
├── docs/
│   ├── development/
│   ├── review/
│   └── superpowers/
└── 项目设计方案/
```

当前不存在的目录应在对应里程碑创建，不应在文档中声明当前已经具备完整一键启动能力。

## 2. 分支和提交

| 类型 | 规范 |
|------|------|
| 功能分支 | `feature/<scope>` |
| 修复分支 | `fix/<scope>` |
| 文档分支 | `docs/<scope>` |
| 提交信息 | `<type>(<scope>): <summary>` |

提交类型建议：`feat`、`fix`、`docs`、`test`、`refactor`、`chore`、`build`。

## 3. 后端编码规范

1. Controller 只处理参数、权限注解和响应转换，不承载业务规则。
2. Service 层必须做对象归属和租户边界校验。
3. Mapper 查询默认受租户插件保护，跨租户查询必须显式标注和审计。
4. DTO、Entity、VO 分离，避免直接暴露数据库实体。
5. 外部输入字段必须校验长度、枚举、范围和格式。
6. 规则引擎运行时结构与数据库结构分离，规则变更后刷新租户级缓存。
7. InfluxDB 查询禁止直接拼接未校验字段名。

## 4. 前端编码规范

1. API 调用集中在 `src/api/`。
2. 权限判断集中在路由守卫、权限指令和 store。
3. WebSocket 订阅必须在页面卸载或变量切换时取消。
4. 图表组件必须释放 ECharts 实例。
5. 仪表盘组件通过注册表声明 type、配置 schema 和渲染组件。
6. 不使用 `v-html` 渲染未净化内容。
7. token 不写入日志和 URL。

## 5. API 变更流程

1. 更新 [05-接口协议说明书](05-接口协议说明书.md)。
2. 更新 DTO 和后端接口。
3. 更新前端 API client。
4. 更新集成测试和 E2E 用例。
5. 如果变更破坏兼容性，需要说明迁移方案。

## 6. 数据库变更流程

1. 新增或修改 MySQL 表必须提交迁移脚本。
2. 新增租户业务表必须包含 `tenant_id` 和租户前缀索引。
3. 修改 InfluxDB tag/field 必须同步修改查询、降采样和测试。
4. Redis key 新增应说明 TTL、命名空间和清理策略。
5. 数据库脚本必须可重复验证，不依赖手工步骤。

## 7. 安全检查清单

| 检查项 | 要求 |
|--------|------|
| 租户隔离 | API、SQL、InfluxDB、WS、Redis key 均有隔离策略 |
| 密钥 | 不写入仓库、镜像、普通配置 |
| token | refreshToken HttpOnly，accessToken 可撤销 |
| 密码 | BCrypt/hash，不记录明文 |
| WebSocket | 连接和订阅都鉴权 |
| CORS/CSRF | 生产限制来源，Cookie 场景启用 CSRF 防护 |
| 审计 | 跨租户、权限、规则和动作配置变更可追踪 |

## 8. 代码评审重点

评审必须优先检查：

1. 是否引入跨租户访问风险。
2. 是否绕过权限校验。
3. 是否破坏 InfluxDB 字段和降采样口径。
4. WebSocket 是否可能向未授权订阅者推送数据。
5. Redis 缓存是否有租户维度和合理 TTL。
6. 敏感信息是否可能进入日志或配置。
7. 测试是否覆盖核心失败路径。

## 9. CI 门禁

主干合并必须满足：

1. 后端编译、单元测试和相关集成测试通过。
2. 前端依赖安装、语法/类型检查和生产构建通过。
3. Markdown 文档链接、格式和关键约束检查通过。
4. 数据库迁移脚本能在空库和已有版本上验证。
5. 密钥扫描未发现密码、token、私钥和生产凭据。
6. 变更涉及接口、数据模型、权限或部署时，对应文档同步更新。

任何绕过 CI 的合并都需要记录原因、影响范围和补测结果。

## 10. 发布前检查

1. `docker compose` 或目标部署环境启动成功。
2. 数据库迁移成功。
3. 健康检查全部通过。
4. 冒烟测试通过。
5. 无明文密钥和测试账号残留。
6. 备份完成并确认回滚方案。
7. 发布版本、镜像 tag、迁移版本和配置版本已记录。

## 11. 文档同步规则

| 变更类型 | 必须同步 |
|----------|----------|
| 新功能 | 需求、接口、测试、实施计划 |
| 数据模型 | 数据文档、迁移脚本、接口文档 |
| 权限变化 | RBAC 文档、接口文档、前端路由 |
| 部署变化 | 部署运维文档、环境变量清单 |
| 协议变化 | 接口协议、前端开发、测试用例 |

文档不是附属物，作为实现和验收依据进入评审。
