# Vionix 低代码仪表盘设计方案

## 1. 概述

拖拽式仪表盘编辑器，用户可通过拖拽组件自定义监控页面，支持图表、指标/状态、告警、表格四类组件，配合全局变量实现组件数据联动，仪表盘支持租户内共享。

## 2. 技术选型

| 项目 | 技术 | 说明 |
|------|------|------|
| 网格布局 | vue-grid-layout | Vue 生态成熟拖拽栅格库，支持拖拽、缩放、自动碰撞 |
| 图表渲染 | ECharts | 复用现有技术栈，实时刷新性能好 |
| 组件注册 | 自建 Map 注册表 | 每种组件 = Vue 文件 + JSON Schema 配置 |
| 数据获取 | 复用现有 WS + InfluxDB API | 不新增数据接口 |

## 3. 编辑器布局

全屏模式，隐藏主框架的侧边导航和顶部栏，最大化画布空间。

```
┌─────────────────────────────────────────────────────────────────┐
│ ← 返回  🏭 仪表盘标题(编辑中)      变量:[设备▼] [时间▼]  预览 保存 │
├────────┬────────────────────────────────────────────┬───────────┤
│ 组件库 │                网格画布                      │ 组件配置   │
│ (可折叠)│         12列 × 无限行                       │ (可折叠)   │
│        │                                             │           │
│ 📈折线图│  ┌──────────────────────┐  ┌──────────┐   │ 标题:     │
│ 📊柱状图│  │   温度实时曲线        │  │  25.3°C  │   │ 数据源:   │
│ 🔵仪表盘│  │   ╱╲  ╱╲             │  │ 当前温度  │   │ 指标:     │
│ 🔢数字指标│  └──────────────────────┘  └──────────┘   │ 粒度:     │
│ 🟢状态卡片│  ┌─────────────┐  ┌─────────────────┐   │ 颜色:     │
│ 🚨告警列表│  │  告警列表    │  │   设备数据表格   │   │           │
```

**交互说明：**

- 从左侧组件库拖入中央画布，自动分配 grid 位置
- 点击画布中组件，右侧加载对应的 schema 配置表单
- 顶部全局变量栏修改变量值，所有引用该变量的组件联动刷新
- 左右面板可通过 ◁ / ▷ 按钮折叠收起

## 4. 数据模型

### 4.1 仪表盘主表 `dashboard`

```sql
CREATE TABLE dashboard (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    tenant_id       BIGINT NOT NULL,
    title           VARCHAR(100) NOT NULL,
    description     VARCHAR(500),
    is_public       BOOLEAN NOT NULL DEFAULT FALSE,
    layout_config   JSON NOT NULL,
    created_by      BIGINT NOT NULL,
    updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_tenant (tenant_id),
    INDEX idx_created_by (created_by)
);
```

### 4.2 `layout_config` JSON 结构

```json
{
  "variables": [
    {
      "name": "device",
      "label": "设备",
      "type": "device",
      "defaultValue": "sensor-001",
      "required": true
    },
    {
      "name": "timeRange",
      "label": "时间范围",
      "type": "timeRange",
      "defaultValue": "-2h",
      "required": true
    }
  ],
  "widgets": [
    {
      "id": "w_001",
      "type": "line-chart",
      "title": "温度实时曲线",
      "grid": { "x": 0, "y": 0, "w": 8, "h": 3 },
      "datasource": {
        "metric": "temperature",
        "device": "$device",
        "level": "realtime",
        "agg": "mean"
      },
      "config": {
        "color": "#4a90d9",
        "smooth": true,
        "areaStyle": true,
        "maxPoints": 120
      }
    }
  ]
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `variables` | 全局变量定义数组，组件通过 `$varName` 引用 |
| `widgets` | 组件实例数组 |
| `widget.id` | 组件唯一标识，前端生成（如 `w_001`） |
| `widget.type` | 组件类型，对应注册表中的 type |
| `widget.grid` | 网格位置：x/y 为坐标，w/h 为列数/行数 |
| `widget.datasource` | 数据源配置，`$device` 引用全局变量 |
| `widget.config` | 组件私有配置，每种组件不同 |

### 4.3 设计要点

| 决策 | 理由 |
|------|------|
| 组件配置存 JSON | 组件类型多样、配置差异大，JSON 比多表关联更灵活 |
| 全局变量存在 JSON 内 | 变量数量少，和仪表盘是一体的 |
| MySQL JSON 类型 | 原生支持 JSON_EXTRACT 按条件查询 |
| 单表设计 | 最小化，后续如需版本管理再加 `dashboard_version` 表 |

### 4.4 查询场景

```sql
-- 某用户可看到的仪表盘（自己的 + 公开的）
SELECT id, title, description, is_public, created_by, updated_at
FROM dashboard
WHERE tenant_id = ?
  AND (created_by = ? OR is_public = TRUE)
ORDER BY updated_at DESC;
```

## 5. 栅格系统

| 参数 | 值 | 说明 |
|------|-----|------|
| 列数 | 12 | 组件最小宽度 = 1/12 屏幕宽 ≈ 8.3% |
| 行数 | 无限 | 组件自动往下排列，画布可纵向滚动 |
| 行高 | 60px | 每行基础高度，组件可跨多行 |
| 间距 | 8px | 组件间间距 |
| 拖拽 | vue-grid-layout | 支持拖拽移动、右下角拖拽缩放、自动碰撞避让 |

## 6. 组件体系

### 6.1 组件类型一览

| 组件 type | 分类 | 最小尺寸 | 说明 |
|-----------|------|---------|------|
| `line-chart` | 图表 | 4x2 | 折线图，支持实时 + 历史 |
| `bar-chart` | 图表 | 4x2 | 柱状图 |
| `area-chart` | 图表 | 4x2 | 面积图 |
| `gauge` | 图表 | 3x2 | 仪表盘（半圆/圆弧） |
| `stat-number` | 指标 | 2x1 | 大数字 + 单位 + 阈值颜色 |
| `status-card` | 指标 | 2x1 | 状态卡片（在线/离线/告警） |
| `device-light` | 指标 | 1x1 | 设备状态灯（绿/红/灰） |
| `alert-list` | 告警 | 4x3 | 告警列表，支持级别过滤 |
| `alert-stats` | 告警 | 3x2 | 告警统计（饼图/柱状图） |
| `data-table` | 表格 | 4x3 | 通用数据表格 |

### 6.2 组件注册机制

```js
// widget-registry.js
const registry = new Map()

function registerWidget(type, { component, schema, defaultProps }) {
  registry.set(type, { component, schema, defaultProps })
}

// 注册示例
registerWidget('line-chart', {
  component: () => import('./widgets/LineChartWidget.vue'),
  schema: {
    title:  { type: 'string', label: '标题' },
    metric: { type: 'metric', label: '指标' },
    device: { type: 'variable', label: '设备', varType: 'device' },
    level:  { type: 'enum', label: '时间粒度', options: ['realtime','min','hour','day'] },
    agg:    { type: 'enum', label: '聚合', options: ['mean','max','min','sum'] },
    color:  { type: 'color', label: '颜色', default: '#4a90d9' },
    smooth: { type: 'boolean', label: '平滑曲线', default: true },
  },
  defaultProps: { color: '#4a90d9', smooth: true, maxPoints: 120 }
})
```

**ConfigPanel 根据 schema 自动渲染配置表单：**

- `string` → 文本输入
- `enum` → 下拉选择
- `color` → 颜色选择器
- `boolean` → 开关
- `metric` → 指标下拉（调 `/api/dashboard-vars/metrics`）
- `variable` → 变量绑定下拉（选择引用哪个全局变量）

### 6.3 数据获取策略

| 粒度 | 来源 | 刷新方式 | 适用组件 |
|------|------|---------|---------|
| 实时 | WebSocket `/ws` | 秒级推送 | line-chart, stat-number, status-card, device-light |
| 分钟/小时/天 | `GET /api/metrics` | 定时轮询 30s | 所有图表类 |
| 告警 | WebSocket + `GET /api/alerts` | 推送 + 轮询 | alert-list, alert-stats |
| 表格 | `GET /api/metrics` | 定时轮询 30s | data-table |

当全局变量变化时，所有引用该变量的组件自动重新请求数据。

## 7. API 设计

### 7.1 仪表盘 CRUD

```
POST   /api/dashboards              创建仪表盘
PUT    /api/dashboards/{id}         更新仪表盘（含布局和组件配置）
DELETE /api/dashboards/{id}         删除仪表盘
GET    /api/dashboards              仪表盘列表
GET    /api/dashboards/{id}         仪表盘详情
PUT    /api/dashboards/{id}/publish 设为公开/私有
```

**创建/更新请求体：**

```json
{
  "title": "车间温度监控",
  "description": "1号车间温湿度实时监控",
  "isPublic": false,
  "layoutConfig": {
    "variables": [...],
    "widgets": [...]
  }
}
```

**列表查询参数：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| pageNum | 页码 | 1 |
| pageSize | 每页条数 | 20 |
| keyword | 标题模糊搜索 | 可选 |

**权限控制：**

| 操作 | 权限 |
|------|------|
| 创建 | 任何已登录用户 |
| 编辑/删除 | 创建者本人 或 租户管理员 |
| 查看 | 自己的 或 `is_public = true` 的 |

### 7.2 全局变量选项接口

```
GET /api/dashboard-vars/devices       设备列表
GET /api/dashboard-vars/metrics       可用指标列表
```

### 7.3 数据查询

复用现有接口，不新增：

- 历史数据：`GET /api/metrics`
- 实时数据：WebSocket `/ws`
- 告警数据：`GET /api/alerts`

### 8.2 路由

```
/dashboards                    列表页（侧边导航新增菜单项）
/dashboards/:id                查看模式（全屏）
/dashboards/:id/edit           编辑模式（全屏）
```

### 8.3 关键流程

**编辑模式：**

```
进入编辑 → 加载 layout_config → 渲染网格 + 组件
  → 拖入新组件 → 自动分配 grid 位置
  → 点击组件 → 右侧加载 schema 表单 → 修改配置 → 实时预览
  → 点击保存 → PUT /api/dashboards/{id}
```

**查看模式：**

```
进入仪表盘 → 加载 layout_config → 渲染变量栏 + 组件（只读）
  → 切换 $device → 所有引用组件重新请求数据
  → 实时组件自动订阅 WebSocket
  → 历史组件每 30s 轮询 InfluxDB
```

**数据流：**

```
全局变量变更
  → useVariableBinding 解析新值
  → 通知所有订阅该变量的组件
  → 各组件根据 datasource 配置获取数据：
      realtime → WebSocket 订阅
      history  → GET /api/metrics
      alert    → GET /api/alerts + WS 监听
```

## 9. 与现有系统集成

| 集成点 | 说明 |
|--------|------|
| RBAC | 仪表盘遵循租户隔离 + 权限控制，新增权限 `menu:dashboard`、`api:dashboard:*` |
| WebSocket | 复用现有 `/ws` 通道，前端按 `$device` + `metric` 过滤消息 |
| InfluxDB | 复用 `GET /api/metrics`，组件将变量解析为参数传入 |
| 规则引擎 | 告警组件复用 `GET /api/alerts` 和 WS 告警推送 |
| 侧边导航 | 新增「仪表盘」菜单项，`menu:dashboard` 权限控制 |
