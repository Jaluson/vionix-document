# Vionix 基础设施与部署方案

## 1. 完整服务组件

| 服务 | 镜像 | 端口 | 职责 |
|------|------|------|------|
| Mosquitto | eclipse-mosquitto:2 | 1883 | MQTT Broker |
| InfluxDB | influxdb:2.7 | 8086 | 时序数据库 |
| Telegraf | telegraf:1.32 | — | 数据采集网关，统一写入 InfluxDB |
| MySQL | mysql:8.0 | 3306 | 关系型数据库，存储规则、告警、设备分组 |
| Redis | redis:7-alpine | 6379 | 缓存，Token 黑名单、登录失败计数、权限缓存 |
| **Backend** | 自建 (Java 17) | 8080 | Spring Boot API + MQTT→WS 桥接 + 规则引擎 |
| **Frontend** | 自建 (Nginx) | 80 | Vue 静态资源 + 反向代理 |

## 3. 基础设施配置

### 3.0 Mosquitto 配置

`mosquitto/config/mosquitto.conf`：

```
listener 1883
allow_anonymous true
# 生产环境建议启用认证：
# password_file /mosquitto/config/passwd
# allow_anonymous false
```

开发环境允许匿名连接以简化调试；生产环境应通过 `password_file` 启用用户名/密码认证。

### 3.1 Telegraf 多数据源配置

所有数据源统一 `name_override = "device_metrics"`，保证降采样 Task 统一覆盖。

```toml
# MQTT（IoT 设备上报）
[[inputs.mqtt_consumer]]
  servers = ["tcp://mosquitto:1883"]
  # topic 格式：vionix/{tenantId}/{deviceId}/telemetry
  topics = ["vionix/+/+/telemetry"]
  data_format = "json"
  name_override = "device_metrics"
  # 从 topic 提取 tenant_id 和 device_id 作为 tag
  [[inputs.mqtt_consumer.topic_parsing]]
    topic = "vionix/+/+/telemetry"
    measurement = "_"
    tags = ["_", "tenant_id", "device_id", "_"]

# Modbus TCP（工业 PLC / 传感器）— 按需启用
# [[inputs.modbus]]
#   name_override = "device_metrics"
#   controller = "tcp://192.168.1.100:502"

# HTTP 推送（应用主动上报）— 按需启用
# [[inputs.http_listener_v2]]
#   name_override = "device_metrics"
#   listen = ":8080"
#   data_format = "json"
```

JSON 结构不要求完全相同，共同字段可跨数据源对比，独有字段各存各的。

### 3.2 MySQL 配置

用于存储规则定义、告警记录、设备分组等结构化数据。

初始化脚本 `mysql/init.sql` 包含所有业务表 DDL：

- **RBAC 模块**（7 张表）：tenant、sys_user、role、permission、user_role、role_permission、token_auth（详见 [RBAC设计方案](RBAC设计方案.md) 第 6 节数据模型）
- **规则引擎模块**（7 张表）：device_group、device_group_member、rule、rule_condition、rule_action、alert、alert_action_log（详见 [规则引擎设计方案](规则引擎设计方案.md) 第 3 节数据模型）
- **仪表盘模块**（1 张表）：dashboard（详见 [低代码仪表盘设计方案](低代码仪表盘设计方案.md) 第 4.1 节）

### 3.3 InfluxDB 降采样 Task 初始化

`influxdb/init.sh` 在 InfluxDB 首次初始化时执行，创建 3 个降采样 Task（min / hour / day），每个 Task 包含 5 种聚合函数（mean / min / max / sum / count）：

```bash
#!/bin/bash
# influxdb/init.sh — 创建降采样所需 Bucket 和 Task
# 注意：此脚本通过 docker-entrypoint-initdb.d 在 InfluxDB 首次 setup 后自动执行

set -e

INFLUX_CMD="influx --host http://localhost:8086 --org vionix --token ${DOCKER_INFLUXDB_INIT_ADMIN_TOKEN}"

# ── 创建降采样 Bucket ──
$INFLUX_CMD bucket create -n device_min -o vionix --retention 2h
$INFLUX_CMD bucket create -n device_hour -o vionix --retention 90d
$INFLUX_CMD bucket create -n device_day -o vionix --retention 0

# ── 1 分钟降采样：device_raw → device_min ──
# 原始字段名为 temperature，降采样后变为 temperature_mean / _max / _min / _sum / _count

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-raw-to-min-mean", every: 1m}

from(bucket: "device_raw")
  |> range(start: -2m)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> aggregateWindow(every: 1m, fn: mean, createEmpty: false)
  |> map(fn: (r) => ({ r with _field: r._field + "_mean" }))
  |> to(bucket: "device_min", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-raw-to-min-min", every: 1m}

from(bucket: "device_raw")
  |> range(start: -2m)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> aggregateWindow(every: 1m, fn: min, createEmpty: false)
  |> map(fn: (r) => ({ r with _field: r._field + "_min" }))
  |> to(bucket: "device_min", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-raw-to-min-max", every: 1m}

from(bucket: "device_raw")
  |> range(start: -2m)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> aggregateWindow(every: 1m, fn: max, createEmpty: false)
  |> map(fn: (r) => ({ r with _field: r._field + "_max" }))
  |> to(bucket: "device_min", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-raw-to-min-sum", every: 1m}

from(bucket: "device_raw")
  |> range(start: -2m)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> aggregateWindow(every: 1m, fn: sum, createEmpty: false)
  |> map(fn: (r) => ({ r with _field: r._field + "_sum" }))
  |> to(bucket: "device_min", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-raw-to-min-count", every: 1m}

from(bucket: "device_raw")
  |> range(start: -2m)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> aggregateWindow(every: 1m, fn: count, createEmpty: false)
  |> map(fn: (r) => ({ r with _field: r._field + "_count" }))
  |> to(bucket: "device_min", org: "vionix")
EOF

# ── 1 小时降采样：device_min → device_hour ──
# 输入字段已带后缀（如 temperature_mean），按后缀分组聚合，输出保持相同后缀

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-min-to-hour-mean", every: 1h}

from(bucket: "device_min")
  |> range(start: -2h)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_mean"))
  |> aggregateWindow(every: 1h, fn: mean, createEmpty: false)
  |> to(bucket: "device_hour", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-min-to-hour-min", every: 1h}

from(bucket: "device_min")
  |> range(start: -2h)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_min"))
  |> aggregateWindow(every: 1h, fn: min, createEmpty: false)
  |> to(bucket: "device_hour", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-min-to-hour-max", every: 1h}

from(bucket: "device_min")
  |> range(start: -2h)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_max"))
  |> aggregateWindow(every: 1h, fn: max, createEmpty: false)
  |> to(bucket: "device_hour", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-min-to-hour-sum", every: 1h}

from(bucket: "device_min")
  |> range(start: -2h)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_sum"))
  |> aggregateWindow(every: 1h, fn: sum, createEmpty: false)
  |> to(bucket: "device_hour", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-min-to-hour-count", every: 1h}

from(bucket: "device_min")
  |> range(start: -2h)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_count"))
  |> aggregateWindow(every: 1h, fn: sum, createEmpty: false)
  |> to(bucket: "device_hour", org: "vionix")
EOF

# ── 1 天降采样：device_hour → device_day ──

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-hour-to-day-mean", every: 1d}

from(bucket: "device_hour")
  |> range(start: -2d)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_mean"))
  |> aggregateWindow(every: 1d, fn: mean, createEmpty: false)
  |> to(bucket: "device_day", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-hour-to-day-min", every: 1d}

from(bucket: "device_hour")
  |> range(start: -2d)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_min"))
  |> aggregateWindow(every: 1d, fn: min, createEmpty: false)
  |> to(bucket: "device_day", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-hour-to-day-max", every: 1d}

from(bucket: "device_hour")
  |> range(start: -2d)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_max"))
  |> aggregateWindow(every: 1d, fn: max, createEmpty: false)
  |> to(bucket: "device_day", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-hour-to-day-sum", every: 1d}

from(bucket: "device_hour")
  |> range(start: -2d)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_sum"))
  |> aggregateWindow(every: 1d, fn: sum, createEmpty: false)
  |> to(bucket: "device_day", org: "vionix")
EOF

$INFLUX_CMD task create - <<'EOF'
option task = {name: "downsample-hour-to-day-count", every: 1d}

from(bucket: "device_hour")
  |> range(start: -2d)
  |> filter(fn: (r) => r._measurement == "device_metrics")
  |> filter(fn: (r) => strings.hasSuffix(v: r._field, suffix: "_count"))
  |> aggregateWindow(every: 1d, fn: sum, createEmpty: false)
  |> to(bucket: "device_day", org: "vionix")
EOF

echo "✓ All buckets and downsampling tasks created."
```

降采样链路：`device_raw` → `device_min` → `device_hour` → `device_day`，共 15 个 Task（3 级 x 5 聚合函数）。

**降采样层级说明：**
- **raw→min**：原始字段添加后缀（如 `temperature` → `temperature_mean`），使用 `map()` 重命名
- **min→hour / hour→day**：按字段后缀过滤聚合（`strings.hasSuffix`），聚合后字段名不变，避免双重后缀
- **_count 聚合**：使用 `fn: sum`（分钟计数求和 = 小时/天总计数）

> **注意**：InfluxDB 的 `docker-entrypoint-initdb.d` 脚本仅在数据卷首次初始化时执行。如果需要在已有数据上补充 Task，可通过 Backend 应用启动时调用 InfluxDB API 创建，或手动执行 `influx task create` 命令。

## 4. 应用层构建

### 4.1 Backend Dockerfile

```dockerfile
# 构建阶段
FROM maven:3.9-eclipse-temurin-17 AS build
WORKDIR /app
COPY pom.xml .
RUN mvn dependency:go-offline -B
COPY src ./src
RUN mvn package -DskipTests -B

# 运行阶段
FROM eclipse-temurin:17-jre
WORKDIR /app
RUN apt-get update && apt-get install -y --no-install-recommends curl && rm -rf /var/lib/apt/lists/*
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 4.3 Frontend Nginx 配置

```nginx
# WebSocket 连接升级映射（放在 http 块中）
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

server {
    listen 80;

    # Vue 静态资源
    location / {
        root   /usr/share/nginx/html;
        index  index.html;
        try_files $uri $uri/ /index.html;   # Vue Router history 模式
    }

    # 反向代理后端 API
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 代理（SockJS/STOMP，含 info 端点和 XHR 回退）
    location /ws {
        proxy_pass http://backend:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
}
```

### 4.4 生产环境 HTTPS 配置

生产环境建议使用 HTTPS，替换上述 4.3 中的 HTTP `server` 块：

```nginx
# 生产环境 HTTPS 配置（替换 4.3 中的 server 块）
server {
    listen 443 ssl;
    ssl_certificate     /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;

    # Vue 静态资源
    location / {
        root   /usr/share/nginx/html;
        index  index.html;
        try_files $uri $uri/ /index.html;
    }

    # 反向代理后端 API
    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # WebSocket 代理
    location /ws {
        proxy_pass http://backend:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
    }
}

# HTTP → HTTPS 重定向
server {
    listen 80;
    return 301 https://$host$request_uri;
}
```

在 `docker-compose.yml` 的 frontend 服务中挂载 SSL 证书：

```yaml
frontend:
  volumes:
    - ./frontend/nginx/ssl:/etc/nginx/ssl:ro
```

## 6. 部署命令


## 6.5 开发环境配置

开发环境中前端（localhost:5173）和后端（localhost:8080）不同源，需要额外配置：

**前端 Vite 代理（推荐）**：

```javascript
// vite.config.js
export default defineConfig({
  server: {
    proxy: {
      '/api': 'http://localhost:8080',
      '/ws': { target: 'http://localhost:8080', ws: true }
    }
  }
})
```

配置后开发环境也是同源访问，`SameSite=Strict` Cookie 正常工作，无需额外处理。

**后端 CORS（备用方案）**：

如果不用 Vite 代理，后端需配置 CORS：
```java
@Configuration
public class CorsConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**")
            .allowedOrigins("http://localhost:5173")
            .allowCredentials(true);
    }
}
```

此时 refreshToken Cookie 需改为 `SameSite=None; Secure`（需 HTTPS 或自签名证书）。

## 7. 服务依赖与启动顺序

```
InfluxDB ──→ init.sh (创建 bucket + task)
MySQL ──→ init.sql (创建规则引擎表)
Redis
   ↓           ↓           ↓
Mosquitto      │           │
   ↓           ↓           ↓
Telegraf ──→ 需要 Mosquitto + InfluxDB
   ↓
Backend  ──→ 需要 Mosquitto + InfluxDB + MySQL + Redis
   ↓
Frontend ──→ 需要 Backend
```

启动后访问：
- **前端页面**：http://localhost
- **后端 API**：http://localhost/api/
- **InfluxDB 管理界面**：http://localhost:8086
- **WebSocket**：ws://localhost/ws
