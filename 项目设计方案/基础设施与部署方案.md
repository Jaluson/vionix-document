# Vionix 基础设施与部署方案

## 1. 阶段边界

当前仓库已经交付的是“可运行最小基础设施”，用于验证 MQTT 接入、Telegraf 写入和 InfluxDB 降采样链路；Backend、Frontend、MySQL、Redis 仍属于目标完整环境的设计内容，只有在对应目录和初始化脚本补齐后才可纳入一键编排。

| 阶段 | 可用组件 | 启动命令 | 可访问能力 |
|------|----------|----------|------------|
| 当前最小环境 | Mosquitto、InfluxDB、Telegraf | `docker compose -f 项目设计方案/docker-compose.yml up -d` | MQTT Broker、InfluxDB UI、MQTT JSON 写入 InfluxDB |
| 目标完整环境 | Mosquitto、InfluxDB、Telegraf、MySQL、Redis、Backend、Frontend | `docker compose -f 项目设计方案/docker-compose.yml up -d --build` | API、Web 前端、RBAC、规则引擎、告警推送、低代码仪表盘 |

当前仓库的部署资产位于 `项目设计方案/`，其中 `docker-compose.yml` 只包含 Mosquitto、InfluxDB、Telegraf。文档中的 MySQL、Redis、Backend、Frontend 配置为目标交付基线，不能视为当前仓库已具备的一键启动能力。

## 2. 服务组件

### 2.1 当前最小环境

| 服务 | 镜像 | 端口 | 职责 |
|------|------|------|------|
| Mosquitto | `eclipse-mosquitto:2` | 1883 | MQTT Broker |
| InfluxDB | `influxdb:2.7` | 8086 | 时序数据库，创建 `device_raw`、`device_min`、`device_hour`、`device_day` |
| Telegraf | `telegraf:latest` | 无 | 数据采集网关，订阅 MQTT JSON 并写入 InfluxDB |

### 2.2 目标完整环境

| 服务 | 镜像 | 端口 | 职责 | 交付前置条件 |
|------|------|------|------|--------------|
| MySQL | `mysql:8.0` | 3306 | RBAC、规则、告警、设备分组、仪表盘元数据 | 补齐 `mysql/init.sql` |
| Redis | `redis:7-alpine` | 6379 | Token 黑名单、登录失败计数、权限缓存 | 多实例/生产环境必需 |
| Backend | 自建 Java 17 | 8080 | Spring Boot API、MQTT 到 WS 桥接、规则引擎、RBAC | 补齐 `backend/` |
| Frontend | 自建 Nginx | 80 | Vue 静态资源、API/WS 反向代理 | 补齐 `frontend/` |

## 4. 统一设备上报模型

基础设施层必须保证 MQTT、InfluxDB、WebSocket 和规则引擎看到同一套设备标识。

### 4.1 MQTT Topic

推荐 topic：

```
vionix/{tenant_id}/{device_id}/metrics
```

兼容 topic：

```
sensors/{device_id}
```

兼容 topic 缺少 `tenant_id` 时，只能用于单租户开发环境，后端和 Telegraf 需要通过默认租户或设备注册表补齐租户信息。

### 4.2 Payload

```json
{
  "timestamp": "2026-05-26T10:01:00Z",
  "source": "mqtt",
  "temperature": 25.3,
  "humidity": 60.5,
  "light": 320
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp` | RFC3339 字符串或 Unix 毫秒 | 可选；缺省使用采集时间 |
| `source` | 字符串 | 数据来源，默认 `mqtt` |
| 业务指标 | 数值/布尔/字符串 | 数值字段进入聚合链路，非数值字段只用于状态或标签扩展 |

### 4.3 InfluxDB Tag 和 Field

| 类别 | 名称 | 来源 | 必选 | 说明 |
|------|------|------|------|------|
| tag | `tenant_id` | topic 或设备注册表 | 是 | 租户隔离边界 |
| tag | `device_id` | topic 或 payload | 是 | 设备过滤和规则匹配主键 |
| tag | `source` | payload 或输入插件 | 是 | `mqtt`、`modbus`、`http` |
| tag | `group_id` | 设备注册表 | 否 | 查询加速，不作为唯一隔离依据 |
| field | 指标名 | payload 业务字段 | 是 | 如 `temperature`、`humidity` |

Telegraf 当前配置已经写入 `device_metrics` measurement，但尚未提取 `tenant_id` 和 `device_id` tag。完整实现时应通过 `topic_parsing`、payload tag 提取或后端写入网关补齐这些 tag，否则历史查询、前端筛选和规则匹配都不能可靠工作。

## 5. 当前最小环境 Compose

当前仓库中的 Compose 应保持最小可运行，不承诺 API 和前端能力：

```yaml
version: "3.8"

services:
  mosquitto:
    image: eclipse-mosquitto:2
    ports:
      - "1883:1883"
    volumes:
      - ./mosquitto/config:/mosquitto/config
      - mosquitto_data:/mosquitto/data
      - mosquitto_log:/mosquitto/log

  influxdb:
    image: influxdb:2.7
    ports:
      - "8086:8086"
    environment:
      - DOCKER_INFLUXDB_INIT_MODE=setup
      - DOCKER_INFLUXDB_INIT_ORG=vionix
      - DOCKER_INFLUXDB_INIT_BUCKET=device_raw
    volumes:
      - influxdb_data:/var/lib/influxdb2
      - ./influxdb/init.sh:/docker-entrypoint-initdb.d/init.sh:ro

  telegraf:
    image: telegraf:latest
    depends_on:
      - mosquitto
      - influxdb
    volumes:
      - ./telegraf/telegraf.conf:/etc/telegraf/telegraf.conf:ro

volumes:
  mosquitto_data:
  mosquitto_log:
  influxdb_data:
```

启动和验证：

```bash
docker compose -f 项目设计方案/docker-compose.yml up -d
docker compose -f 项目设计方案/docker-compose.yml ps
docker compose -f 项目设计方案/docker-compose.yml logs -f telegraf
```

当前阶段可访问：

- InfluxDB 管理界面：`http://localhost:8086`
- MQTT Broker：`localhost:1883`

当前阶段不可访问：

- 后端 API：尚未交付 `backend/`
- 前端页面：尚未交付 `frontend/`
- WebSocket：尚未交付后端 WS 服务

## 6. 目标完整环境 Compose 增量

完整环境需要在当前 Compose 基础上增加以下服务。该片段只有在对应目录和脚本存在后才能启用：

```yaml
  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
      - MYSQL_DATABASE=vionix
      - MYSQL_USER=vionix
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./mysql/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data

  backend:
    build:
      context: ./backend
    ports:
      - "8080:8080"
    environment:
      - MQTT_BROKER=tcp://mosquitto:1883
      - INFLUXDB_URL=http://influxdb:8086
      - INFLUXDB_ORG=vionix
      - INFLUXDB_BUCKET=device_raw
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/vionix?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=Asia/Shanghai
      - SPRING_DATASOURCE_USERNAME=vionix
      - SPRING_DATASOURCE_PASSWORD=${MYSQL_PASSWORD}
      - SPRING_DATA_REDIS_HOST=redis
      - SPRING_DATA_REDIS_PORT=6379
      - AUTH_TOKEN_HASH_SALT=${AUTH_TOKEN_HASH_SALT}
      - AUTH_JWT_SECRET=${AUTH_JWT_SECRET}
    depends_on:
      - mosquitto
      - influxdb
      - mysql
      - redis

  frontend:
    build:
      context: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

volumes:
  mysql_data:
  redis_data:
```

生产部署中不得把数据库密码、JWT 密钥、Token 哈希盐、RSA 私钥等敏感配置写死在 Compose 或普通 `application.yml` 中，应使用环境变量、Docker Secret、KMS 或平台 Secret 管理。

## 7. 服务依赖与启动顺序

```
InfluxDB -> init.sh 创建 bucket 和降采样 task
Mosquitto -> Telegraf 订阅 MQTT 并写入 InfluxDB
MySQL -> init.sql 创建 RBAC、规则、告警、仪表盘表
Redis -> 提供多实例共享安全状态
Backend -> 依赖 Mosquitto、InfluxDB、MySQL、Redis
Frontend -> 依赖 Backend API 和 WebSocket
```

单实例开发环境可临时用本地缓存替代 Redis；多实例、生产环境和需要“登出立即失效/登录失败全局锁定”的环境必须部署 Redis。
