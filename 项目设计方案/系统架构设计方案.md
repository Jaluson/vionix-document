# Vionix 系统架构设计方案

## 1. 项目概述

IoT 设备监控平台，支持多数据源（MQTT / Modbus / HTTP）采集、多粒度存储、实时监控与历史查询，内置规则引擎实现实时告警检测与动作分发。

## 3. 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 实时推送 | SockJS + STOMP over WebSocket | Spring 内置 STOMP 支持，SockJS 兼容降级 |
| MQTT 客户端 | Eclipse Paho / Spring Integration MQTT | 订阅 Mosquitto，同时供规则引擎实时评估 |
| InfluxDB 查询 | influxdb-client-java | 官方 Java SDK，支持 Flux 查询 |
| MySQL | mysql-connector-java + MyBatis-Plus | 规则/告警/分组存储，JPA 自动建表可选 |
| 规则引擎 | 自建内存引擎 | 规则编译缓存 + 匹配索引 + 滑动窗口状态跟踪 |
| 前端图表 | ECharts | 实时刷新性能好，数据量大不卡 |
| 前端 WS | @stomp/stompjs + sockjs-client | 秒级数据接收 + 告警推送，SockJS 自动降级 |
| 容器化 | Docker Compose + 多阶段构建 | 前后端统一编排，一键部署 |
